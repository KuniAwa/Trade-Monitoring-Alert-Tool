# 相場監視アラートツール 仕様書

## 1. 概要

- **目的**: 為替（USD/JPY・EUR/JPY 等）と日経225系シンボルを15分ごとに監視し、条件成立時にメールで通知する。平日夜に **1日1回の相場サマリー** メールを送る。
- **実行環境**: Vercel（Python Runtime）、15分間隔の Cron。
- **データソース**:
  - **為替**: Twelve Data API（日足・15分足・1時間足）。
  - **日経225**: Yahoo Finance Chart API（15分足・1時間足・参照用日足）。前日高安の算出に Twelve Data をフォールバックとして使う場合あり。
- **通知**: メールのみ。自動売買は行わない。

## 2. 前提条件

| 項目 | 内容 |
|------|------|
| Vercel | Pro 以上（Cron 利用） |
| 実行間隔 | 15分（`*/15 * * * *`） |
| シークレット | APIキー・メール設定は Environment Variables で管理 |
| タイムゾーン | Cron は UTC。OHLC の取得・表示は `Asia/Tokyo` を主に使用 |

## 3. 監視対象

| 銘柄 | データ取得 | 備考 |
|------|------------|------|
| USD/JPY | Twelve Data `USD/JPY` | 既定で監視 |
| EUR/JPY | Twelve Data `EUR/JPY` | 既定で監視 |
| AUD/JPY | Twelve Data `AUD/JPY` | コード上 `SYMBOLS_DISABLED` で停止可能 |
| 日経225 | Yahoo `NIY=F` / `^N225` 等 | 環境変数 `NIKKEI_SYMBOL` で明示可能。未設定時は取得可能な候補を自動選択 |

## 4. 足種・参照高安

| 用途 | 足種 | タイムゾーン・備考 |
|------|------|-------------------|
| エントリー判定 | **15分足** | `Asia/Tokyo` |
| トレンド（環境認識） | **1時間足** | `Asia/Tokyo`。Twelve Data は `outputsize=120`（EMA50 用）。Yahoo は `60m` + 十分なレンジ |
| 為替の参照高安 | **日足** | **NY 基準**（`America/New_York`）。直近2本のうち **1本前** の high / low を「前日高値・前日安値」とする |
| 日経の参照高安 | 15分足から集計 | **前営業日・JST 日中 09:00〜15:45** の high / low。Yahoo で算出し、失敗時は日足フォールバックや Twelve Data 15分足を試行 |

## 5. アラート条件（すべて満たしたときのみ通知）

判定対象は **直近確定の15分足**（API 上は未確定足を避け、確定済みの1本前の足を使用）。

### 5.1 1時間足トレンド（環境認識）

直近 **確定した1時間足** の終値と、同一足時点の **EMA（終値ベース、アプリ内計算）** を用いる。

- **上昇環境（ロングの前提）**  
  - `終値 > EMA(20) > EMA(50)`  
  - かつ **EMA(20)** が直近3本（確定足を含む）で **単調増加**（傾き上向き）
- **下降環境（ショートの前提）**  
  - `終値 < EMA(20) < EMA(50)`  
  - かつ **EMA(20)** が直近3本で **単調減少**

EMA の期間は定数 `EMA_FAST_1H=20` / `EMA_SLOW_1H=50`（`api/cron.py`）。

### 5.2 ブレイク条件（ATR バッファ付き）

15分足で **Wilder ATR(14)** を算出し、バッファ `BREAKOUT_ATR_MULT × ATR`（既定 `0.20 × ATR`）を用いる。

- **ロング**: 確定15分足の **終値 > 前日高値 + バッファ**
- **ショート**: 確定15分足の **終値 < 前日安値 − バッファ**

「前日高・前日安」の定義は **4. 参照高安** に従う（為替＝NY 日足、日経＝JST 日中セッション）。

### 5.3 初ブレイク（同一閾値の連続発火を抑える）

確定足の **1本前の確定足** の終値が、当該方向の閾値をまだ越えていないこと。

- **ロング**: `前足終値 ≤ 前日高値 + バッファ` かつ `今足終値 > 前日高値 + バッファ`
- **ショート**: `前足終値 ≥ 前日安値 − バッファ` かつ `今足終値 < 前日安値 − バッファ`

15分データが3本未満の場合は初ブレイク条件を満たさない（通知なし）。

### 5.4 15分足 20MA 条件

- **ロング**: 確定終値 **>** 15分足 SMA(20)（終値ベース）
- **ショート**: 確定終値 **<** SMA(20)

### 5.5 パラボリック SAR（オプション）

定数 `USE_PARABOLIC_SAR_IN_ALERTS` が **True** のときのみ条件に含める（既定は **False**）。

- **ロング**: 終値 > 当該足の SAR
- **ショート**: 終値 < SAR  

SAR は high / low / close からアプリ内計算（Twelve Data の指標 API は使わない）。

### 5.6 押し率フィルタ

ブレイク局面からの戻りを **押し率（％）** で評価し、**50% 以上** のときは通知しない（定数 `OSHIRITSU_EXCLUDE_PCT`）。33% 以内を理想とし、メール本文に注記することがある（`OSHIRITSU_IDEAL_PCT`）。

### 5.7 通知メールに含めるリスクシナリオ（参考）

条件成立時のメールに、**参考**として以下を併記する（自動注文は行わない）。

- **ストップ候補①**: 前日高（ロング）／前日安（ショート）
- **ストップ候補②**: `終値 ± STOP_ATR_MULT × ATR(15分)`（既定 `1.5 × ATR`）。ATR が算定できない場合は①のみ
- **採用ストップ**: ①と②のうち **エントリー価格に近い（タイトな）方**
- **リスク幅**: エントリー終値と採用ストップの距離
- **利確目安**: 1R・2R（リスク幅の1倍・2倍を終値から加減算）

## 6. 重複通知の防止（簡易設計）

- **同一 Cron 実行内** で、同一銘柄・同一方向のメールは **1通まで**（実装は `seen` セットで重複排除）。
- **日をまたぐ重複**を防ぐには、Vercel KV / Postgres 等に「最後に通知した日付（JST）+ 銘柄 + 方向」を保持する方式が望ましい。本リポジトリの MVP では未実装のため、**初ブレイク条件** により同一ブレイク局面での連続発火は抑えられるが、押し戻り後の再ブレイクでは再度通知され得る。

---

## 7. 相場サマリー（1日1回）

- **送信タイミング**: 平日 JST **23:00〜23:14** の Cron 実行時（ずれ対策のウィンドウ）。
- **内容**: 監視対象各銘柄について、確定15分足・参照高安・1時間 EMA 環境・ATR・ブレイク閾値・初ブレイク相当の真偽・押し率（ロング/ショート想定）・**現在終値を前提とした**ロング/ショート別のリスク参考（採用ストップ・リスク幅・1R/2R）を **1通のメール** にまとめる。
- アラート条件の成立とは独立（スナップショット表示）。

---

## 8. システム構成

```
Vercel Cron (*/15 * * * *)
    → GET /api/cron
        → Authorization: Bearer <CRON_SECRET> で検証
        → settings.json の監視時間内なら各銘柄について:
            1. 参照高安の取得（為替: Twelve Data 日足 NY / 日経: Yahoo 等）
            2. 15分足・1時間足の取得（銘柄に応じ Twelve Data または Yahoo）
            3. SMA・EMA・ATR・（任意）SAR・押し率・トレンド判定
            4. 成立時のみアラートメール（同一実行内の銘柄×方向は1回）
        → 平日 23:00 台ならサマリー用スナップショットを組み立て、1通送信
```

Twelve Data の **Minutely レート制限** 対策として、実装では **FX 処理後に一定秒数空けてから日経** を処理する場合がある。

## 11. API 利用方針・指標の計算

- **Twelve Data**（為替中心）: `time_series`（15min / 1h / 1day）。指標専用 API は使わず、必要な系列は **アプリ内で算出**。
- **Yahoo Finance**（日経）: Chart API（15m / 60m / 1d 等）。
- **アプリ内計算**:
  - **SMA(20)**（15分終値）
  - **EMA(20), EMA(50)**（1時間終値）
  - **ATR(14)** Wilder 方式（15分 high / low / close）
  - **パラボリック SAR**（任意・既定オフ）

1銘柄あたりの Twelve Data リクエスト数は、銘柄・キャッシュ・フォールバックにより変動する（日足＋15分＋1時間が基本）。

## 12. メール本文（概要）

- **アラート**  
  - 件名: `[相場アラート] {銘柄名} 前日{高値|安値}ブレイク（ロング|ショート） - {日時}`  
  - 本文: 15分（終値・20MA・ATR・ロング/ショート閾値）、参照高安のラベルと価格、1時間足（終値・EMA20/50）、押し率、ブレイク後高/安、**リスクシナリオ（ストップ・1R/2R）**。
- **サマリー**  
  - 件名: `[相場サマリー] {銘柄数}銘柄定時レポート - {日時}`  
  - 本文: 上記に沿ったスナップショットを銘柄ごとに列挙。
