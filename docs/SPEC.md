# 相場監視アラートツール 仕様書（MVP）

## 1. 概要

- **目的**: USD/JPY と 日経225先物 の価格を15分ごとに監視し、条件成立時にメールで通知する。
- **実行環境**: Vercel（Python Runtime）、15分間隔のCron。
- **データソース**: Twelve Data API。
- **通知**: メールのみ。自動売買は行わない。

---

## 2. 前提条件

| 項目 | 内容 |
|------|------|
| Vercel | Pro 以上（Cron 利用） |
| 実行間隔 | 15分（`*/15 * * * *`） |
| シークレット | APIキー・メール設定は Environment Variables で管理 |
| タイムゾーン | Cron は UTC。必要に応じて Asia/Tokyo で表示・判定 |

---

## 3. 監視対象

| 銘柄 | Twelve Data シンボル（想定） | 備考 |
|------|------------------------------|------|
| USD/JPY | `USD/JPY` | 為替 |
| 日経225先物 | `N225` または要確認 | 指数・先物は Twelve Data のシンボル一覧で要確認 |

※ 日経225先物のシンボルは Twelve Data の「Symbol Search」やサポートで確認し、環境変数で上書き可能とする。

---

## 4. 足種・データ

- **メイン足**: 15分足（`interval=15min`）、表示は日本時間（`timezone=Asia/Tokyo`）
- **前日高値・前日安値**: 日足（`interval=1day`）の直近2本のうち、1本前の日足の high / low を使用。**NY 基準**のため `timezone=America/New_York` で取得（Twelve Data の 1day はタイムゾーン指定でその地域の「日」で集計される）

---

## 5. アラート条件（すべて満たしたときのみ通知）

### 5.1 ブレイク条件（いずれか1つ）

- **前日高値ブレイク**: 直近確定15分足の終値（close）が、前日高値（previous day high）を上回る。
- **前日安値ブレイク**: 直近確定15分足の終値が、前日安値（previous day low）を下回る。

### 5.2 20MA条件（15分足）

- **ロング方向（前日高値ブレイク時）**: 直近確定15分足の終値 > 15分足20MA（SMA, 終値ベース）。
- **ショート方向（前日安値ブレイク時）**: 直近確定15分足の終値 < 15分足20MA。

※ 20MA は Twelve Data の SMA 指標（`time_period=20`, `series_type=close`）を使用。

### 5.3 パラボリックSAR条件

- **ロング方向**: 直近の SAR が価格の下側（買いシグナル）であること。  
  つまり、直近確定15分足の終値 > 直近の SAR 値。
- **ショート方向**: 直近の SAR が価格の上側（売りシグナル）であること。  
  つまり、直近確定15分足の終値 < 直近の SAR 値。

※ SAR は Twelve Data の Parabolic SAR エンドポイント（`/sar`）を使用。  
  acceleration / maximum はデフォルト（0.02 / 0.2）でよい。

### 5.4 まとめ（通知が飛ぶパターン）

1. **前日高値ブレイク** かつ **終値 > 20MA** かつ **終値 > SAR** → ロング条件でメール通知。
2. **前日安値ブレイク** かつ **終値 < 20MA** かつ **終値 < SAR** → ショート条件でメール通知。

## 6. 重複通知の防止（簡易設計）

- **同一銘柄・同一方向・同一「日」** に1回まで通知する。
- 「日」の定義: **Asia/Tokyo の日付**（0:00〜24:00）で判定。
- 実装方法（例）:
  - Vercel の Environment Variables または KV/Blob 等は使わず、**ファイルベース**は Vercel 上では永続化できないため、**メモリ不可**。
  - **簡易案**: 直近 N 本の15分足の「日付（Asia/Tokyo）」と「ブレイク方向」をキーに、**同じ日付で同じ方向の通知を1回だけ送る**ようにする。  
    ただし Vercel Serverless はステートを持たないため、**「直近1本の15分足」のみを評価**し、  
    **「その15分足の始まりの時刻（Asia/Tokyo）の日付」をキーに、同じ日付・同じ銘柄・同じ方向の通知を送ったかどうかを、外部に持たずに防ぐのは困難**。
- **MVP での簡易設計**:
  - **「直近2本の15分足」を取得**し、**直近1本目（確定足）** で条件判定する。
  - 同じ Cron 実行内では、**同一銘柄・同一方向の通知を1回だけ送る**（同一実行内の重複防止）。
  - **日をまたぐ重複防止**は、**Vercel KV**（Pro で利用可）や **Vercel Postgres**、または外部の軽量ストア（例: 1キーだけの Redis）に「最後に通知した日付（Asia/Tokyo）+ 銘柄 + 方向」を書き、読みに使う。  
    **MVP では「同一 Cron 実行内の重複防止」のみ実装**し、日をまたぐ重複は「15分ごとの実行なので最大でも1日あたり数回」とし、許容するか、または README で「Vercel KV を導入すると日単位の重複防止が可能」と記載する。

※ 本実装では、**同一実行内の重複防止**と、**「前回通知した時刻」を環境変数やレスポンスからは参照しない**シンプルな形とし、**日跨ぎ重複防止はオプション（Vercel KV のキー例のみ仕様に記載）**とする。

## 7. システム構成

```
Vercel Cron (*/15 * * * *)
    → GET /api/cron
        → Authorization: Bearer <CRON_SECRET> で検証
        → 各銘柄について:
            1. Twelve Data: 日足で前日 high/low 取得
            2. Twelve Data: 15分足のみ取得（20MA と パラボリックSAR はアプリ内で計算）
            3. 条件判定（ブレイク + 20MA + SAR）
            4. 成立時のみメール送信（同一実行内で同一銘柄・同一方向は1回）
```

## 8. 環境変数

| 変数名 | 必須 | 説明 |
|--------|------|------|
| `CRON_SECRET` | ○ | Vercel Cron 用。16文字以上推奨。`Authorization: Bearer <CRON_SECRET>` で照合。 |
| `TWELVE_DATA_API_KEY` | ○ | Twelve Data API キー。 |
| `ALERT_MAIL_FROM` | ○ | 送信元メールアドレス（SMTP の認証アカウントと一致させる想定）。 |
| `ALERT_MAIL_TO` | ○ | 通知先メールアドレス（複数可の場合はカンマ区切り等、実装で定義）。 |
| `SMTP_HOST` | ○ | SMTP サーバーホスト。 |
| `SMTP_PORT` | ○ | SMTP ポート（例: 587）。 |
| `SMTP_USER` | ○ | SMTP 認証ユーザー。 |
| `SMTP_PASSWORD` | ○ | SMTP 認証パスワード。 |
| `NIKKEI_SYMBOL` | - | 日経225先物のシンボル。未設定時は `N225` を使用。 |

## 10. Twelve Data API 利用方針・指標の計算

- **Twelve Data で取得するもの**
  - **時系列（15分足）**: `GET /time_series?symbol=...&interval=15min&outputsize=...`
  - **日足（前日 high/low）**: `GET /time_series?symbol=...&interval=1day&outputsize=3`
- **アプリ内で計算するもの**（Twelve Data の指標APIは使わない）
  - **SMA（20）**: 15分足の終値から Simple Moving Average（20本）を算出。
  - **パラボリックSAR**: 15分足の high / low / close から Wilder の標準アルゴリズム（AF=0.02, max=0.2）で算出。

※ 1回の Cron あたり、銘柄2 × **2リクエスト**（日足 + 15分足のみ）のため、API クレジット・レート制限の負荷が軽い。Grow プランで SAR 等の指標APIが不要になる。

## 11. メール本文（例）

- 件名: `[相場アラート] {銘柄名} 前日{高値|安値}ブレイク - {日時}`
- 本文: 銘柄、方向（ロング/ショート）、条件成立時刻（Asia/Tokyo）、終値・前日高値/安値・20MA・SAR の概略を記載。
